# 规范合规性审查提示词模板

调度规范合规性审查子代理时使用此模板。

**目的：** 验证实现者构建了被请求的内容（不多不少）

```
Task 工具 (general-purpose):
  description: "审查任务 N 的规范合规性"
  prompt: |
    你正在审查一个实现是否匹配其规范。

    ## 请求了什么

    [需求任务的完整文本]

    ## 实现者声称构建了什么

    [来自实现者的报告]

    ## 关键：不要信任报告

    实现者完成得可疑地快。他们的报告可能不完整、不准确或过于乐观。你必须独立验证一切。

    不要：
    - 相信他们关于实现了什么的说法
    - 相信他们关于完整性的声明
    - 接受他们对需求的解释

    要做：
    - 阅读他们写的实际代码
    - 逐行比较实际实现与需求
    - 检查他们声称实现但未实现的部分
    - 寻找他们未提及的额外功能

    ## 你的工作

    阅读实现代码并验证：

    缺失需求：
    - 是否实现了所有被请求的内容？
    - 是否有跳过或遗漏的需求？
    - 是否声称某事工作但未实际实现？

    额外/不需要的工作：
    - 是否构建了未被请求的东西？
    - 是否过度工程或添加不必要的功能？
    - 是否添加了规范中没有的"锦上添花"？

    误解：
    - 是否以不同于意图的方式解释需求？
    - 是否解决了错误的问题？
    - 是否以错误的方式实现了正确的功能？

    通过阅读代码验证，而不是相信报告。

    报告：
    - ✅ 规范合规（代码检查后一切匹配）
    - ❌ 发现问题：[具体列出缺失或多余的内容，含 file:line 引用]
```
