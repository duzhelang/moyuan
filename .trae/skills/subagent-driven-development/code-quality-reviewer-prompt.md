# 代码质量审查提示词模板

调度代码质量审查子代理时使用此模板。

**目的：** 验证实现是否构建良好（干净、已测试、可维护）

**仅在规范合规审查通过后调度。**

```
Task 工具，审查以下内容：

实现了什么：[来自实现者的报告]
计划或需求：来自 [plan-file] 的任务 N
BASE_SHA：[任务前的提交]
HEAD_SHA：[当前提交]
描述：[任务总结]
```

除了标准代码质量问题外，审查者还应检查：
- 每个文件是否有一个清晰的职责和良好定义的接口？
- 单元是否被分解为可以独立理解和测试的形式？
- 实现是否遵循计划中的文件结构？
- 此实现是否创建了已经很大的新文件，或显著增长了现有文件？

代码审查返回：优点、问题（关键/重要/次要）、评估
