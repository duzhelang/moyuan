## 设计规范

### 技术栈
- Vue 3 + Element Plus 2.4.4 UI 组件库
- ECharts 5.6.0 数据可视化
- 图标使用 `@element-plus/icons-vue`

### 色彩体系
- 主色调: Element Plus 默认主题色（#409EFF）
- 功能色: 成功（#67C23A）、警告（#E6A23C）、危险（#F56C6C）、信息（#909399）
- 背景色: 浅色模式 #F5F7FA，深色模式 #1A1B1D

### 排版规范
- 字体: 系统默认无衬线字体
- 字号层级: 12px(辅助) / 14px(正文) / 16px(标题小) / 18px(标题中) / 20px+(标题大)
- 行高: 1.5倍字号

### 间距规范
- 内边距: 8px / 12px / 16px / 20px / 24px / 32px
- 外边距: 8px / 12px / 16px / 20px / 24px / 32px
- 组件间距: 16px（默认）

### 圆角规范
- 小圆角: 4px（按钮、输入框）
- 中圆角: 8px（卡片、弹窗）
- 大圆角: 16px（大卡片、对话框）

## 设计原则

### 一致性
- 同一类组件保持相同的样式和行为
- 遵循Element Plus的设计语言
- 保持页面间的布局一致性

### 清晰性
- 信息层级分明，重要内容突出
- 操作路径清晰，减少用户学习成本
- 错误提示明确，指导用户修正

### 效率
- 常用操作放在显眼位置
- 减少不必要的点击步骤
- 批量操作提供快捷方式

### 美观
- 保持适当的留白，避免拥挤
- 色彩搭配协调，不刺眼
- 字体大小和颜色对比度符合可访问性标准

## 工作流程

### 1. 需求分析
- 理解功能需求和用户场景
- 确定页面结构和信息层级
- 参考Element Plus组件库选择合适的组件

### 2. 界面设计
- 设计页面布局和组件排布
- 确保响应式适配
- 保持设计一致性

### 3. 交互设计
- 设计用户操作流程
- 定义交互反馈（loading、hover、点击效果）
- 考虑异常状态处理

### 4. 视觉审查
- 检查色彩、字体、间距是否规范
- 检查组件对齐和间距
- 检查不同屏幕尺寸的适配

## 约束条件
- 所有输出使用简体中文
- 优先使用Element Plus内置组件
- 不引入额外UI框架
- 保持深色/浅色主题适配
