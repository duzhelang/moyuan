# 项目特定技能触发规则

| 技能 | 触发条件 |
|------|----------|
| **oda-project** | 询问项目结构、功能模块、技术细节、业务逻辑时 |
| **data-assets** | 询问数据文件位置、训练集内容、模型存储、数据归档、目录结构时 |
| **ui-design-system** | 新增页面、调整样式、统一视觉风格时 |
| **ui-implementer** | 根据设计规范实现具体UI组件时 |
| **analysis-planner** | 用户提出新需求、功能开发、问题排查等复杂任务时，进行需求分析和任务分解 |
| **developer** | 涉及代码编写时调用，通用开发代理，要求先熟读开发规范 |
| **code-reviewer** | 代码编写完成后，依据规范检查代码合规性 |
| **doc-manager** | 代码变更后，检查并更新 `docs` 目录下的相关文档 |
| **local-fullstack-tester** | 测试本地前后端服务联通性时，测试前自动读取 `.trae/agents/local-fullstack-tester/AGENT.md` 和 `.trae/test-env.yml` |

## 触发优先级
1. 用户明确指定技能 → 立即调用
2. 匹配触发条件 → 自动调用
3. 无匹配技能 → 直接执行任务

## 禁止滥用
- 不要为了调用技能而调用，必须与当前任务强相关
- 同一任务不要重复调用相同技能
- 技能是辅助工具，任务完成才是目标
